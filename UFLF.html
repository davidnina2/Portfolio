L'UNION FAIT LA FORGE

Ce jeu a pour but de montrer aux joueurs que la diversité est importante et parfois nécessaire. Pour ce faire, les joueurs s'affrontent dans un match en 1v1 où ils doivent recruter des employés et utiliser des cartes utilitaires pour obtenir l'avantage sur leur adversaire. Pour encourager la diversité, il y a trois races disponibles : les Gobelins, les Amphibiens et les Ogres. Chaque race a ses propres types d'expertise, donc il est essentiel de faire un bon mélange, sinon notre forge ne décrochera aucun contrat, ce qui entraînera notre défaite.

Objectif du jeux

Les objectifs du jeu étaient principalement éducatifs, mais voici certains objectifs et éléments que nous souhaitions que notre jeu possède :

  • - Un mode multijoueur 1v1
  • - Une boucle 100% jouable
  • - Des animations et des modèles interessant à regarder
  • - Qu'il soit facile et rapide à prendre en main

Mes Contributions

Le fonctionnement backend du jeux

Puisque j'étais l'un des seuls programmeurs, j'ai réalisé la grande majorité du code backend, ce qui signifie que j'ai pris en charge tous les éléments pour la gestion de chaque phase du jeu et le traitement des données qui y sont liées. Cela m'a permis d'apprendre à créer un code optimisé et facile à lire. Pour ce jeu, j'ai principalement utilisé les DataTable d'Unreal Engine pour importer toutes les informations concernant les cartes disponibles que nous utilisons dans le jeu. De plus, j'ai également fait usage des composants Timeline pour gérer les interactions visuelles. Ce jeu m'a aussi aidé à mieux comprendre les différentes classes d'Unreal, telles que le Player Controller, le Player State, le GameMode, etc., dans le contexte d'un jeu multijoueur.
